# FilingDelta

FilingDelta 是一个**面向 A 股、H 股和中概股的可追溯信披分析工具**。

它的第一版核心目标，不是做一个泛化的“AI 投研助手”，而是聚焦在一个更窄、更实用的场景上：

> **帮助用户非线性阅读定期报告。**

很多投资者、研究员或重度财报读者，虽然并不熟悉 AI 工具，但又需要长期、大量阅读公司披露文件来辅助投资判断。  
FilingDelta 面向的正是这类用户：它不试图替代用户做投资决策，而是帮助用户**更快找到定期报告中最重要、最值得先看的部分**，并把这些信息组织成可追溯的结构化结果。

FilingDelta 的基本工作方式是：

- 从公开披露材料中提取关键信息
- 将冗长的定期报告压缩成更适合快速阅读的结构化摘要
- 帮助用户优先查看最关键的指标、表述变化和风险信息
- 支持通过超链接或引用，快速跳转回原始文件中的对应位置
- 在“先看重点、再回原文核对”的流程中，提升阅读效率

FilingDelta 关注的是**公开信披材料的提取、摘要、引用定位和变化检测**，而不是投资建议。

## 适用用户

FilingDelta 当前主要面向这样一类用户：

- 不太会使用 AI 工具
- 但需要广泛、大量阅读信披材料
- 希望更快完成财报、公告和定期报告的初步阅读
- 希望先抓住重点，再回到原文进行核对
- 最终仍然由自己完成判断和决策

换句话说，FilingDelta 不是“替你做判断”的工具，而是一个**帮助你更高效阅读定期报告的辅助工具**。

## 第一版核心价值

FilingDelta 第一版最重要的能力是：

### 非线性阅读定期报告
用户不必从头到尾顺序阅读数十页甚至上百页的定期报告，而可以先从 FilingDelta 提取出的结构化结果中快速把握重点，再根据引用和超链接回到原始文件核对细节。

这种阅读方式适合处理：

- 篇幅很长的年报和半年报
- 同时需要看多家公司的定期报告
- 需要快速定位重点段落、关键表格和风险变化的场景

## 当前 Demo 状态

当前仓库已经落下了一个**在本地配好样例文档后即可运行、可演示**的第一版单文档 demo，重点证明这条主链路成立：

1. 导入一份公开信披文档
2. 运行单文档分析 workflow
3. 生成结构化重点摘要
4. 抽取关键财务字段并绑定出处
5. 在三栏界面中查看原文、摘要、关键数字与 citation detail
6. 对缺少可靠证据的结果显式标记人工复核状态

当前 demo 已经实现的重点包括：

- `FastAPI` 后端 demo API
- `LlamaIndex Workflows` 单文档 workflow
  - `Orchestrator`
  - `Reader`
  - `FactExtractor`
  - `Verifier`
- 本地 parse + structured extraction 主链路
- `React + TypeScript + Vite` 三栏阅读界面
- PDF 连续阅读、缩放、跳页与 citation 联动
- `通过 / 需复核 / 失败` 的 review status 展示
- 右栏 `Ask Filing`：
  - document-scoped RAG
  - Mixed QA
  - chat memory
  - step-level telemetry
- typed evidence retrieval 的第一轮演进：
  - `section_text` 用于叙述型 / 风险 / 战略类问题
  - `table_row` 用于指标 / 表格类问题
  - `page_text` 继续作为 citation anchor 和 fallback

当前 demo 以**单文档可追溯阅读**为主；双文档 `Compare / diff` 能力仍在下一阶段推进。

## 支持的材料范围

FilingDelta 当前主要面向以下公开披露材料：

- A 股年报、半年报、业绩预告、业绩快报、问询函回复
- H 股年报、业绩公告及相关披露文件
- 中概股 / ADR 的 20-F、6-K、业绩材料等

## 核心能力

FilingDelta 长期聚焦四类核心任务：

### 1. 重点提取
- 提取关键财务指标
- 提取管理层讨论与分析中的重点内容
- 提取风险因素及重要表格信息
- 从长篇报告中提炼最值得优先阅读的部分

### 2. 结构化阅读
- 生成 1–2 页的结构化摘要
- 帮助用户快速浏览报告核心内容
- 将重点信息整理成更易于扫描和核查的形式

### 3. 引用定位
- 为每条结论附上原文出处
- 支持回到原始文件中的对应位置
- 支持“先看摘要，再查原文”的阅读流程

### 4. 对比与变化检测
- 对比今年与去年披露差异
- 对比不同市场之间的披露差异
- 对比公告、年报、业绩材料之间的信息一致性
- 标记需要人工复核的潜在异常点

其中，当前公开仓库已稳定实现的是：

- 单文档重点提取
- 结论级 citation 回溯
- 关键财务字段抽取与出处绑定
- 单文档 review status / 人工复核提示

双文档差异分析仍然属于下一步扩展。

## 运行 Demo

### 环境要求

- Python 3.12
- `uv`
- Node.js `20.16+` 或 `22.3+`

### 后端

```bash
uv sync --dev
uv run uvicorn filingdelta.main:app --reload
```

### 前端

```bash
cd frontend
npm install
npm run dev
```

默认地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

在 Windows 本地环境下，也可以直接使用仓库根目录下的启动脚本：

- `start_demo.cmd`
- `start_backend.cmd`
- `start_frontend.cmd`

### 当前 demo 界面

第一版界面采用三栏工作台：

- 左栏：文档概览、关键数字、重点摘要、review status
- 中栏：原始文档 viewer 与 citation detail
- 右栏：`Ask Filing` 单文档问答与 Mixed QA

当前中栏重点支持：

- PDF 连续阅读
- 缩放与适应宽度
- 跳页
- 点击摘要/数字后联动查看 citation detail

### 样例数据说明

公开仓库默认**不附带真实样例文档**。本地 demo 依赖：

- `data/raw/` 下的本地样例文件
- `data/raw/small_doc_benchmark.json` 中登记的文档 manifest

也就是说，远端克隆下来后，代码和前端都可以直接启动，但要真正看到样例文档列表并运行 demo，需要先在本地补好样例文件和对应 manifest。原始样例与解析产物默认不提交到远端仓库。

## 项目定位

FilingDelta 是一个**公开信披材料分析与阅读辅助工具**，用于帮助用户更高效地阅读、核对和比较公司披露文件。

它**不提供**：
- 投资建议
- 股票推荐
- 评级
- 目标价
- 组合配置建议

本项目的所有输出仅用于信息整理、阅读辅助、研究支持与文档分析，用户应自行核验原始披露材料并独立作出判断。
